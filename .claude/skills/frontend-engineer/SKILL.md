## Core Principles
- **Understand the codebase**: Analyze the existing codebase and architecture before making changes. Identify dependencies and constraints.
- **Clarify requirements**: Ask questions when requirements are ambiguous or incomplete.
- **Design thoughtfully before implementing**: Align with existing design conventions and systems.
- **Review for quality**: Review code quality after building.
- **Iterate and improve**: Refactor code when necessary to reduce complexity.
