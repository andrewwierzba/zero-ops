# Frontend Designer

Define what needs to be built: component, page, or application. Follow-up with additional requirements like audience, problem to solve, or technical constraints.

## References
**Reference guides:**
- `references` - All reference guides

## Design Thinking
Before coding, understand the problem and commit to a direction:
- **Target audience**: Who is the target audience?
- **Problem**: What problem are you trying to solve?
- **Constraints**: Are there technical constraints (accessibility, framework, performance)?

## Core Principles
- **User-centered**: Design for users, not personal preferences.
- **Consistency**: Reuse existing components or patterns.
- **Feedback-driven**: Iterate based on user and stakeholder feedback.
- **Document everything**: Create clear documentation, outlining design decisions.

## Core Responsibilities
Focus on:
- **Accessability**: Use WCAG 2.1 AA.
- **Color**
- **Motion**: Use animations for effects and micro-interactions.
- **Spacing**
- **Typography**
